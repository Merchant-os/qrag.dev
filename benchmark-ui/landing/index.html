RAG vs Baseline

Understanding the two approaches

Baseline Agent

A pure generation approach that relies solely on the model's internal knowledge without any external context or examples.

  • No retrieval of similar examples
  • Only uses system prompt context
  • Faster response time
  • Good for simple tasks
No Context

RAG Agent

A retrieval-augmented approach that finds and includes relevant Q# examples from a curated corpus before generating code.

  • Semantic search for similar examples
  • Includes top-k examples in context
  • Uses OpenAI embeddings (text-embedding-3-small)
  • Better for complex quantum algorithms
Context-Aware

How Metrics Are Measured

Real compilation and test execution

Unlike synthetic benchmarks that estimate correctness, this system actually builds generated Q# code and runs xUnit tests to verify functional correctness. Each metric reflects real execution results.

pass@1

Pass @ 1

The percentage of tasks where the first generated code sample passes all xUnit tests. This measures reliability of single-shot generation.

Tasks with passing first code / Total tasks
Why it matters: In production scenarios, developers typically use the first suggestion without regenerating. High pass@1 means the AI is consistently accurate.
pass@3

Pass @ 3

The percentage of tasks where any of the 3 generated code samples passes all xUnit tests. This measures generation quality with multiple attempts.

Tasks with ≥1 passing code / Total tasks
Why it matters: Shows the AI's capability range and how often it can produce correct code given multiple attempts. Useful when regenerating is acceptable.
compile@1

Compile @ 1

The percentage of tasks where the first generated code sample successfully compiles (even if tests fail). This measures syntactic correctness.

Tasks with compilable first code / Total tasks
Why it matters: Compilation is a basic requirement. High compile@1 but low pass@1 indicates the AI generates syntactically valid code with logical errors.

Measurement Process

1. Generate Code

AI generates Q# code for each benchmark task (3 attempts per task)

2. Build Project

Generated code is added to Q# project and compiled using .NET SDK

3. Run Tests

xUnit tests execute to verify correctness of generated quantum operations

4. Calculate Metrics

Results aggregated into pass@1, pass@3, and compile@1 scores

Knowledge Corpus

Where the examples come from

The RAG system uses a curated corpus of 40-70 high-quality Q# examples scraped from Microsoft's Quantum Development Kit (QDK) repository. These examples are indexed using OpenAI's text-embedding-3-small model for semantic search.

The corpus includes:

  • Basic quantum operations (gates, measurements, superposition)
  • Quantum algorithms (Grover's search, phase estimation, etc.)
  • Error correction schemes
  • Quantum teleportation protocols
  • GHZ and Bell state generation

Examples are deduplicated and selected for quality, ensuring the RAG system retrieves relevant, working code patterns to aid generation.
